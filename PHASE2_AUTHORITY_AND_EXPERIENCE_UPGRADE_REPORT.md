# Phase 2 Authority And Experience Upgrade Report

## 1. Selection Experience Changes

- Replaced inline Spiritkin preview selection behavior with a fullscreen reveal overlay.
- Clicking a founder card now opens a dominant fullscreen stage instead of relying on a cramped inline preview.
- When an intro trailer exists, it becomes the primary selection surface:
  - large
  - centered
  - visually dominant
  - no competing still art rendered beside it
- The fullscreen reveal includes:
  - Spiritkin name
  - title / domain context
  - explicit confirm selection action
- After confirmation, the chosen founder is preserved in a cleaner selection summary block rather than an inline trailer-driven layout.

## 2. SpiritCore Greeting Flow Changes

- Added a backend-backed SpiritCore greeting route: `POST /v1/spiritcore/welcome`.
- After SpiritGate, the flow now stops in a dedicated SpiritCore welcome stage.
- If the user has not provided a name yet:
  - SpiritCore requires the name there
  - founder selection does not proceed until the name is submitted
- Once the name is submitted:
  - SpiritCore greeting text is generated by backend route logic
  - frontend speaks/displays that backend greeting
  - only then does the app proceed to founder selection
- Conversation bootstrap was also updated so first conversation greetings can include the user name.

## 3. Grand Stage Background Enforcement

- Bound active game theming to the currently selected Spiritkin domain instead of relying on generic or mixed presentation.
- Added Spiritkin-domain theme resolution on the frontend so active room and Grand Stage visuals follow the active bonded companion.
- Added explicit archive-domain handling for Elaria instead of letting that path collapse into generic crown visuals.
- Generic fallback remains only as failure safety, not as the intended authority path.

## 4. Trailer System Cleanup

- Removed selection trailer dependence from the older inline `buildBondPreview` path.
- Intro trailers are now treated as authority media for the fullscreen selection stage only.
- Bonded home and ordinary support media surfaces now stay on still/canonical art instead of trying to reuse selection trailers.
- Side panels and media helper usage remain still-art based.

## 5. Navigation Improvements

- Simplified top-level room navigation from tab-heavy naming into action-first intent labels:
  - Speak
  - Play
  - Reflect
  - Explore
- These actions still map to the same Phase 1 room structure:
  - Speak -> Presence Room
  - Play -> Games Room
  - Reflect -> Journal Room
  - Explore -> Events Room
- Functionality was preserved while intent and clarity were improved.

## Verification Notes

- Verified modified frontend and backend files with `node --check`.
- Verified targeted authority hooks exist for:
  - fullscreen selection overlay
  - SpiritCore backend greeting route
  - Spiritkin-domain game theming
- Full browser-level experiential verification still depends on runtime/manual UI pass.
